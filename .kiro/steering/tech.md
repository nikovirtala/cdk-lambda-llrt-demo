# Technology Stack

## ⚠️ Projen-Managed Project

This project is **entirely managed by projen** using the `AwsCdkTypeScriptAppProject` template from `@nikovirtala/projen-constructs`. Key implications:

- **DO NOT** manually edit generated files (package.json, tsconfig.json, etc.)
- **ALL** configuration changes must be made in `.projenrc.ts`
- After modifying `.projenrc.ts`, run `npx projen` to regenerate files
- Dependencies, scripts, and tooling are defined in `.projenrc.ts`, not package.json

## Project Template
- **Template**: `AwsCdkTypeScriptAppProject` from `@nikovirtala/projen-constructs`
- **Purpose**: Provides opinionated setup for AWS CDK TypeScript projects with modern tooling
- **Features**: Includes Biome, Vitest, Homebrew/mise integration, and CDK deployment automation

## Build System
- **Projen**: Project configuration and build orchestration (all scripts run through `npx projen`)
- **pnpm**: Package manager
- **TypeScript**: Primary language (v5.9.3)
- **ESM**: ECMAScript modules (not CommonJS)

## AWS Infrastructure
- **AWS CDK**: v2.165.0+ for infrastructure as code
- **LLRT**: Low Latency Runtime for Lambda functions
- **cdk-lambda-llrt**: Construct library for LLRT functions

## Code Quality
- **Biome**: Linting and formatting (replaces ESLint/Prettier)
  - 4-space indentation
  - Double quotes for strings
  - 120 character line width
- **Vitest**: Testing framework with snapshot support

## Runtime Requirements
- Node.js >= 24.11.1
- ARM64 architecture for Lambda deployment

## Common Commands

### Development
```bash
pnpm install          # Install dependencies
npx projen build      # Compile TypeScript
npx projen test       # Run tests
npx projen test:watch # Run tests in watch mode
```

### CDK Operations
```bash
npx projen synth      # Synthesize CloudFormation
npx projen diff       # Show infrastructure changes
npx projen deploy     # Deploy to AWS
npx projen destroy    # Tear down stack
```

### Code Quality
```bash
npx projen biome      # Run linter and formatter
```

### Maintenance
```bash
npx projen upgrade    # Upgrade dependencies
npx projen            # Regenerate projen files
```

## Important Notes
- All configuration files marked with "Generated by projen" should not be edited directly
- Modify `.projenrc.ts` and run `npx projen` to regenerate configuration
- Use `.ts` extensions in imports (TypeScript with `allowImportingTsExtensions`)
